##### H5
Reflection on the entire process of the game. 
It took me a while to get to the main idea that I wanted to develop for this basic game text but in the end I managed to developing the concept thanks to specific references, like Dungeons and Dragons, older RPG classic games, such as The Elder Scrolls III Morrowind or even simple decision-making games, like Fallout New Vegas. I used Claude AI in terms of developing the concepts and to understand the entire system of conditional statement but I managed to write the code on my own, with specific feedback with Claude AI in terms of fixing, modifying the code when there was specific amount of mistakes. Particularly, when placing the corresponding inputs and how much separation space was necessary in between the different segments of the code. I tried not to be dependent on the AI as I wanted to develop my own personal programming and designing skills, since I have always been interested in making my own games. The very first concept of Dungeon Exploration was kind of shallow. It was not intended to have three specific routes. In the very first begining, I thought of only making two main routes based on melee combat and stealthy. I even thought about adding extra elements, such as magic, potions, health restoration and so on but managed to discard those elements since made no actual sense for the simple text game. Also, I had no clear idea how to implement these elements in order to make the game function properly. The idea is the following:
1) Left route: is basically based on the use of strenght. In which you have to defeat an enemy. I wanted to introduce a goblin, which is the basic enemy in most of the RPG games or even MMORPGs (such as RuneScape, which is also a huge inspiration for this game). But afterwards I decided to introduce an orc to make the experience much more challenging. If you fail in this part, you literally just die and could not get the treasure. Therefore, the entire quest/adventure has to be considered as a huge failure
2) Right route: is mostly inspired on The Hobbit. The dragon sleeping is basically a reference to Smaug in the Tolkienverse. The main part of this entire area is to sneak through in order to get the treasure. If you make noise, you literaly die. The dragon will awaken and your character will eventually colapse to the neverending fire of it. I did not add this part as a print but it is suggested in the way the game tells you that you have failed. 
For the center route, ffter a long while fixing and adding narrative aspects in order to make the player's experience much more engaging and interactive, I considered the idea of introducing the center route as part of the game. In the beginning I had the clear intention of introduction a sphynx, which is a clear reference to the puzzles that Harry Potter had to solve during the Goblet of Fire book in the last part of the Tournament Trials. But this idea had problems in terms of being feasible. How is it possible to place a sphynx in the middle of a dungeon? So I discarded the idea and added a strange magician that is a resemblence of the ancient world. The entire game is based on my personal storytelling project and I intend to add more elements to it. For the two riddles, I had to look them up on Google as I did not want to make the game excesively confusing. I had a lot of fun though but since this is my actual first practice of creating a simple game (not counting the mini inividual coding project which was basically made by AI), I did not want to make it extremely complex. 
After a long struggle with fixing the codes, I had to change the entire prints since those were in input mode and could not be showed as part of the narrative. After a long while, I had so many problems while trying to make it run on a Python file that I had to consult Claude AI in order to fix those problems. Nevertheless, after this process I managed to make it run as a Python file and after revising some of the outcomes it ended up working. The usage of Claude AI was merely intented for guidance and as a feedback for improvement, not to copy the code entirely so I had to write over and over again until it worked but in the final revision, after a couple of problematic issues while trying to run the game, this extra help ended up being very helpful. Hopefully, I can keep learning more on how to program these games in order to be fully able to do this on my very own without as I would love also to design my very own independent games based on decision-making, adventure, fantasy and exploration. I wish I could have added more aspects in terms of the lore or worldbuilding but since it is my very first attempt hopefully the game ends up being engaging :)
